Introducción

A continuacion te presentamos una guia corta de todos los comandos que necesitas para manejar las versiones de tu proyecto con la herramienta Git. En la presente guia no se explicara cada uno de los conceptos que git maneja, solo se mostrara sus comandos y cual es su funcionalidad.

Primeros Pasos

Una vez finalizada la instalacion de Git, abriremos la "Git Bash" y haremos unas configuraciones basicas de inicio.


Configuración de Nombre de Usuario

git config --global user.name "TuNombreAqui"

Configuracion de Correo Electronico

git config --global user.email "TuCorreo@Aqui.com"

Ver Lista de todas las configuraciones

git config --global --list

Comandos para gestionar tu proyecto

Para iniciar un proyecto primero debes de posicionarte en la carpeta de este.

cd "Ruta\A\Tu\Carpeta"

Comando de Inicio

Con esto git registrara la carpeta actual como un proyecto, ahora esta lista para hacer el manejo de versiones.

git init

Preparar un archivo para el commit

git add "NombreDeArchivo.extencion"

Agregar todos los archivos y carpetas para el commit

git add -A

Ver el estado de tu proyecto

git status

Hacer un commit y incluir un mensaje

git commit -m "Mensaje Descriptivo de la version"

Ver lista de Commits o versiones

git log

Crear un archivo txt con la lista de commits

git log > NombreDeArchivo.txt

Regresar a una version anterior de tu proyecto

git checkout CodigoSHA

Posicionarte en la version mas reciente

git checkout master

Crear una nueva rama

git branch NombreDeLaRama

Eliminar una rama

git branch -D NombreDeLaRama

Fusionar dos Ramas

Antes de hacer esto, asegurate de posicionarte en la rama principal, la cual absorvera todos los cambios

git merge NombreDeLaRama
